第45-65行：add()方法 - 添加消息到记忆

@Override
public void add(String conversationId, List<Message> messages) {
    try {
        // 1. 先读取已有记录
        List<MsgRecord> records = readRecords(conversationId);
        
        // 2. 将新消息转为MsgRecord格式，加入列表
        for (Message m : messages) {
            records.add(MsgRecord.from(m));
        }

        // 3. 判断是否超出窗口
        if (records.size() > maxMessages) {
            // 超出部分的数量
            int overflow = records.size() - maxMessages;
            // 取出超出的旧消息（列表开头部分）
            List<MsgRecord> old = records.subList(0, overflow);
            // 将旧消息追加到摘要中
            appendSummary(conversationId, old);
            // 截断列表，只保留最新的maxMessages条
            records = records.subList(overflow, records.size());
        }

        // 4. 将截断后的消息写回Redis
        writeRecords(KEY_PREFIX + conversationId, records);
    } catch (Exception e) {
        log.error("Redis 写入记忆失败", e);
    }
}

@Override
public List<Message> get(String conversationId) {
    // 读取记录
    List<MsgRecord> records = readRecords(conversationId);
    // 如果为空返回空列表
    if (records.isEmpty()) return List.of();
    // 转换为Spring AI的Message对象返回
    return records.stream().map(MsgRecord::toMessage).toList();
}

第91-108行：appendSummary()方法 - 追加消息到摘要
-----------------------------------------------------------------------
private void appendSummary(String conversationId, List<MsgRecord> overflow) {
    StringBuilder sb = new StringBuilder();
    
    // 先读取已有摘要，追加到前面
    String existing = redisTemplate.opsForValue().get(SUMMARY_PREFIX + conversationId);
    if (existing != null && !existing.isEmpty()) {
        sb.append(existing).append("\n");
    }
    
    // 将每条溢出消息格式化为"用户：xxx"或"AI：xxx"
    for (MsgRecord r : overflow) {
        String label = "USER".equals(r.role) ? "用户" : "AI";
        sb.append(label).append("：").append(r.content).append("\n");
    }
    
    // 摘要截断，最多2000字符，防止无限膨胀
    String text = sb.toString();
    if (text.length() > 2000) {
        text = text.substring(text.length() - 2000);
    }
    
    // 写入Redis，设置30分钟过期
    redisTemplate.opsForValue().set(SUMMARY_PREFIX + conversationId, text, TTL_MINUTES, TimeUnit.MINUTES);
    log.debug("摘要已更新，长度={}", text.length());
}

第110-120行：readRecords()方法 - 从Redis读取记录
-----------------------------------------------------------------------
private List<MsgRecord> readRecords(String conversationId) {
    // 读取JSON字符串
    String json = redisTemplate.opsForValue().get(KEY_PREFIX + conversationId);
    if (json == null || json.isEmpty()) return new ArrayList<>();
    
    try {
        // JSON反序列化为MsgRecord列表
        return objectMapper.readValue(json,
                objectMapper.getTypeFactory().constructCollectionType(List.class, MsgRecord.class));
    } catch (JsonProcessingException e) {
        log.error("Redis 记忆反序列化失败", e);
        return new ArrayList<>();
    }
}

第122-129行：writeRecords()方法 - 将记录写入Redis
-----------------------------------------------------------------------
private void writeRecords(String key, List<MsgRecord> records) {
    try {
        // 序列化为JSON字符串
        String json = objectMapper.writeValueAsString(records);
        // 写入Redis，设置30分钟过期
        redisTemplate.opsForValue().set(key, json, TTL_MINUTES, TimeUnit.MINUTES);
    } catch (JsonProcessingException e) {
        log.error("Redis 记忆序列化失败", e);
    }
}

第131-155行：MsgRecord内部类 - 消息记录（用于序列化）
-----------------------------------------------------------------------
public static class MsgRecord {
    public String role;    // 角色：USER / ASSISTANT / SYSTEM
    public String content; // 内容
    
    public MsgRecord() {}  // 无参构造（Jackson需要）
    
    public MsgRecord(String role, String content) {
        this.role = role;
        this.content = content;
    }
    
    // 从Spring AI的Message转换为MsgRecord
    static MsgRecord from(Message m) {
        if (m == null) return new MsgRecord("USER", "");
        return new MsgRecord(m.getMessageType().name(), m.getText());
    }
    
    // 从MsgRecord转换回Spring AI的Message
    Message toMessage() {
        if (content == null) return new UserMessage("");
        // 根据role创建不同类型的Message
        return switch (role) {
            case "ASSISTANT" -> new AssistantMessage(content);
            case "SYSTEM" -> new SystemMessage(content);
            default -> new UserMessage(content);
        };
    }
}

为什么要用MsgRecord而不是直接序列化Message？
→ Message是接口，Spring AI的实现类有复杂的内部结构，Jackson反序列化会失败。
→ MsgRecord只有role和content两个字段，简单易序列化。

private String buildSystemPrompt(String sessionId, List<Document> docs) {
    // 获取历史摘要
    String summary = redisMemory.getSummary(sessionId);

    StringBuilder sb = new StringBuilder();
    sb.append("你是超市智能导购「小智」。请基于以下信息回答用户问题。回答简洁亲切，控制在 150 字以内。\n\n");

    // 如果有摘要，先插入摘要
    if (!summary.isEmpty()) {
        sb.append("---历史摘要---\n").append(summary).append("\n");
    }

    // 再插入检索到的商品信息
    sb.append("---商品信息---\n");
    for (int i = 0; i < docs.size(); i++) {
        sb.append("【").append(i + 1).append("】").append(docs.get(i).getText()).append("\n");
    }

    return sb.toString();
}

五、整体流程总结
=======================================================================

用户发送消息 → AI回复的完整流程：

1. 用户发送消息
   → ChatOrchestrator.chat()
   → RouterService.classify() 判断意图
   → 分发到 RagAgentService 或 ChatServiceImpl

2. 获取历史摘要
   → redisMemory.getSummary(sessionId)
   → 从Redis读取 chat:summary:{sessionId}

3. 拼接消息
   → 如果有摘要："【历史摘要】xxx\n【用户当前问题】xxx"
   → 如果无摘要：直接使用用户消息

4. 调用LLM
   → RagAgentService：摘要 + 检索结果 → System Prompt
   → ChatServiceImpl：摘要 + 用户消息 → User Message

5. 保存对话记录
   → PromptChatMemoryAdvisor 自动调用 ChatMemory.add()
   → RedisChatMemory.add() 写入Redis
   → 超出10条时，旧消息移到 chat:summary

6. TTL过期清理
   → 所有Redis Key设置30分钟TTL
   → 超时自动删除，清理僵尸会话

=======================================================================
六、设计亮点
=======================================================================

1. 接口兼容：实现ChatMemory接口，业务代码零改动
2. 双Key存储：chat:memory存完整消息，chat:summary存溢出摘要
3. Last N + Summary：既保留最近上下文，又不丢失历史
4. 摘要截断：2000字符上限，防止Prompt无限膨胀
5. TTL自动清理：30分钟过期，无需手动维护
6. 序列化友好：MsgRecord简单结构，解决Message接口反序列化问题
